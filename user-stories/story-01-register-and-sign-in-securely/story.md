# User Story 1

## User Story
As a patient, I want to register and sign in securely so that I can access the portal.

## Story Points
8 story points

## Acceptance Criteria
### Patient
- Patients should be able to create an account using the required registration details.
- The system should validate required fields and prevent duplicate accounts for the same email address.
- Passwords should be handled securely and should not be shown or stored in plain text.
- Patients should be able to sign in using valid credentials and access the portal.
- Invalid sign-in attempts should show a clear error message without exposing sensitive information.
